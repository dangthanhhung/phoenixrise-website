The Breakthrough

MPC that survives compromise

Traditional MPC wallets assume key shares stay safe forever. When enough shares are stolen — through patience, phishing, or insider threat — the security model collapses permanently. Phoenix Chain automatically refreshes shares with hardware-backed entropy. Even if all shares are stolen, future operations remain secure.

Post-Compromise Security for MPC

Phoenix Injection provides hardware-backed entropy that automatically rotates MPC key shares. Compromised shares become mathematically worthless within one epoch. No manual intervention. No downtime. The system heals itself.

7-Layer Security Architecture

Hardware root of trust → TEE isolation → encrypted storage → epoch key rotation → PCS injection → Merkle commitment → quarantine enforcement. Attack one layer, six more stand. Defense at every level of the stack.

Automatic Recovery

Compromised key shares are detected through anomalous signing patterns, quarantined immediately, and replaced with fresh shares backed by hardware entropy. Zero human intervention required.
