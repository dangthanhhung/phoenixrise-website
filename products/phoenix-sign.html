Deep Dive

Core concepts explained

Epoch-Based Signing

No permanent private keys exist. Each epoch generates a fresh keypair: seed_epoch = HKDF(VaultKey ∥ sigma ∥ ctx). Key used once, then destroyed. Compromise of one epoch cannot affect any other. Forward secrecy + post-compromise recovery.

Deterministic Key Derivation

Every signing key is fully deterministic — derived from hardware secret + evolving state + message policy. Reproducible for audit. Unpredictable without VaultKey. DyLWE operates on R_q = Z_q[X]/(X^N+1) using Learning With Rounding for deterministic noise.

Merkle Key Commitment

All epoch public keys committed to a Merkle tree. Verifiers only need root hash + current epoch proof for O(log n) verification. Compact, efficient, cryptographically binding.

Phoenix Refresh — The PCS Invariant

VaultKey ← HKDF(VaultKey ∥ new_secret, "phoenix"). After refresh, every future seed, key, and signature changes completely. An attacker with full RAM + storage snapshot before refresh is permanently locked out. This is the invariant: every hash, KDF, PRF, and state evolution MUST depend on the current VaultKey.

Quarantine & Anti-Rollback

On reboot, suspected compromise, or sync loss: quarantine activates. No real payloads signed — only pings and dummy operations. Monotonic counters prevent state rollback attacks. System waits for hardware-confirmed fresh entropy before resuming.

GenericPCSWrapper

Algorithm-agnostic interface wraps any PQC algorithm in PCS guarantees within one day. FormalPCSModel provides mathematical proofs. KUSS Framework validates security properties with 5 axioms and 6 theorems. 100% formal verification pass rate.
