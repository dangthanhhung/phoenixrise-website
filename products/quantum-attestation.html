Technology

Seven layers of breakthrough technology

Each innovation reinforces the others. Together they create the most comprehensive attestation system in digital asset security.

DualSign — Two Algorithms, One Attestation

ML-DSA (FIPS 204) signs in ~2ms using lattice-based mathematics. SLH-DSA (FIPS 205) adds ~50ms using only hash functions — the most conservative cryptographic assumption known. Both must pass verification. If a mathematical breakthrough breaks one, the other stands.

Epoch Keys — No Permanent Private Keys

Traditional systems protect one key forever. Phoenix generates a fresh keypair for every signing epoch. Each key is derived deterministically from hardware-backed state, used exactly once, then permanently destroyed. A compromised epoch key reveals nothing about past or future keys.

EphemeralSelf-Destroying

Post-Compromise Security via Phoenix Injection

The critical property no competitor offers. After any breach, hardware-backed entropy (VaultKey in TEE/Secure Enclave) is injected into the key derivation flow. Every future key changes completely. An attacker with full system access before injection has zero knowledge of future states.

FAQ

Common questions

Does Quantum Attestation replace my existing signatures?
No. It's a pure overlay. Your ECDSA/EdDSA signing infrastructure stays untouched. Phoenix adds quantum-safe attestation on top — creating a separate, independently verifiable proof layer that will survive when classical algorithms are broken.
Why two algorithms instead of one?
ML-DSA and SLH-DSA use fundamentally different mathematics — lattices vs. hash functions. A breakthrough breaking one wouldn't affect the other. In cryptography, single-algorithm risk costs billions. DualSign eliminates it at the cost of 50ms.
What's the performance impact?
ML-DSA: ~2ms. SLH-DSA: ~50ms. Total under 100ms including network. Transaction settlement is measured in seconds to minutes — attestation is invisible. Storage: ~11KB per dual attestation, trivial against the value being protected.
Can anyone verify, or do I need Phoenix?
Anyone can verify independently. The verifier is open-source. You need only the attestation data and the public root key. No connection to Phoenix, no API key, no subscription required for verification.
What about NIST standard updates?
Algorithm Agility is built in. GenericPCSWrapper can integrate new PQC algorithms within one day. When NIST finalizes additional standards, Phoenix adapts without infrastructure changes, without downtime, without migration projects.
